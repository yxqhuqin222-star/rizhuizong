# 项目规则

## 数据

- `tongji_demo.xlsx` 是每日现状数据，`tongji_target.xlsx` 是每周目标数据。
- 指标字段、公式、最新期次和状态判断以 [PRD](../docs/PRD-progress-dashboard.md) 为唯一口径。
- 两个来源不一致时先暴露差异，不能静默选值。
- 客户证据、实际数据和已记录决策优先于内部推测；推测必须明确标记。

## 交付

- 完成标准不仅是代码可运行，还包括最终工作簿、网页接口和播报图使用相同口径。
- 上线或交付前运行相关测试，并通过 `/api/health` 和目标接口确认当前进程已加载最新代码。
- 公式修改必须用真实样例写出完整算式和预期结果，再同步修改代码、PRD、最终产物和回归测试。
- 发布后至少用一次真实 `demo` 更新完成端到端验证。

## 文档

- PRD 只保留当前有效需求和计算口径。
- CHANGELOG 按日期记录已完成的重要变化，不承担当前规则说明。
- `decisions/` 保存仍值得追溯的架构选择。
- `source/` 保存原始需求证据，不编辑历史原文。
