# AGENTS.md — 日追踪看板

## 沟通与执行

- 默认使用中文，先给结论，再说明必要过程。
- 先检查现有项目和上下文；只有缺失信息会实质改变结果时才询问。
- “检查、分析、审查、查原因”默认只读；“修复、实现、安装、部署”可执行完成任务所需的范围内操作。
- 删除数据、回滚用户改动、提交、推送、发布或修改外部账号前，必须取得明确授权。
- 用户要求改变已确认的方案时，先列出拟变更和风险，确认后再实施。

## 修改原则

- 只实现用户要求的最小范围，不顺手重构或清理无关内容。
- 修改前确认目录并检查未提交改动；保留用户已有修改。
- 修改计算逻辑前，核对 [PRD](./docs/PRD-progress-dashboard.md)、最终输出和当前代码。口径不一致时先说明，不直接选一处修改。
- 公式变更必须同步更新计算代码、PRD 中的规则、最终生成物和自动化测试。
- 本地服务修改后，重启或热加载服务，并调用真实健康检查或目标接口确认新代码已加载。

## 验证标准

- 小改动运行能证明目标行为的最小检查。
- Bug 修复先确认根因或复现条件，再验证问题消失。
- 工作簿和报表必须执行完整生成链路，并读取最终产物验证；不能只看中间 CSV、HTML 或脚本输出。
- 最终回复说明完成内容、验证结果、涉及文件和未覆盖检查。
