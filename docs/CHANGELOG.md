# Changelog — 日追踪看板

所有重要功能变更都记录在这里。最新变更放在最上方。

## 2026-06-28

### Added

- 自然语言查询支持通过渠道简称匹配 `last_from`，首批包含 SH、YZY、万C、YYB、RQ、JJ。
- 查询缺少日期、渠道或指标时进行限定式多轮追问，最多两轮，不使用默认值猜测。
- 查询结果增加真实明细、10 / 20 / 50 条分页、条件解释和完整明细导出。

### Changed

- 自然语言查询从浏览器输入框升级为页面内独立查询工作区。
- 渠道名称与 `last_from` 映射保存在可审计的本地 CSV，不引入数据库或外部模型。

### Verified

- 已验证 `YZY渠道的进量` 会追问时间，补充 `6月27日` 后返回 538 单并命中 538 行。
- 已验证原有“日期 + `last_from` + 成单量”查询继续可用。
- 已验证未知渠道和同时出现多个渠道时不会猜测结果。

## 2026-06-27

### Changed

- 浏览器标签新增蓝色“日”字站点图标。
- 看板和播报图片中的年级按业务顺序展示：小学二至六年级、初中初一至初三、高中高一至高三。
- 将原“正常”状态进一步细分：完成率比进度高 10 个百分点及以上时显示为“快”，并同步应用于看板和播报图片。

## 2026-06-25

### Added

- 搭建本地网页看板 V1，用于查看 `tongji_demo.xlsx` 和 `tongji_target.xlsx` 生成的进度监控数据。
- 支持上传每日 `demo` 文件、更新每周 `target` 文件，并重新生成 summary。
- 默认展示每个 `学部` 的最新 `期次` 数据，历史期次可通过筛选查看。
- 增加 KPI 卡片：最新期次目标、现状、完成率、平均进度、落后项。
- 增加筛选能力：`学部`、`期次`、`线索渠道二级分类`、`价体`、关键词。
- 增加导出能力：导出最新期次、导出完整 Summary、导出查询明细。
- 增加自然语言查询入口，V1 支持按日期和 `last_from` 查询 `成单量`。
- 增加每日进度播报图导出，分别生成小学、初中、高中三张 PNG 图片。
- 增加每日进度播报图钉钉群播报，支持将小学、初中、高中播报图通过成单机器人发送到群里。
- 增加 `lec1元占比` 播报图，按小学最新期次 1 元数据统计指定 last_from 尾号渠道的成单量、目标占比和实际占比。

### Changed

- 时间进度公式调整为 `(当前日期 - 进量日期 - 1) / 总天数`，总天数仍为 6 天，结果限制在 `0%` 到 `100%`。
- summary 字段统一为业务中文字段：`学部`、`期次`、`线索渠道二级分类`、`价体`、`年级`、`下单日期`、`成单量`。
- `线索渠道二级分类` 中以 `外部微转-` 开头的分类统一汇总为 `外部微转-*`。
- 主表展示顺序调整为：`学部`、`期次`、`线索渠道二级分类`、`价体`、`年级`、`target_time`、`目标`、`现状`、`差距`、`完成率`、`进度`、`状态`。
- 主表不再展示 `下单日期`，但保留它用于日期筛选。
- 时间进度按 `当前日期 - target 表进量日期` 计算，并限制在 `0%` 到 `100%`。
- 自然语言查询改为低频入口，放到导出区域底部，不占用主监控区域。
- 明确播报图字段口径：`渠道展示 = 线索渠道二级分类 + 价体`，`招生目标 = 目标`，`剩余天数 = 总天数 - (target_time - 进量日期)`。
- 播报图只展示小学、初中、高中各自最新一期次数据，不展示历史期次。
- summary 工作簿的“数据概览”增加默认展示口径、最新期次范围、最新期次指标和播报图字段口径；维度分布改为围绕最新期次 summary 展示 `目标` 和 `现状`。
- 播报图生成时会强制规范化 summary 中的日期和数值字段，避免新底表导出后出现字符串与空值混合导致生成失败。
- 最新期次口径改为以 `target` 表中的期次为准；demo 中存在但 target 中不存在的更晚期次保留在完整 Summary，不进入默认看板和播报图。
- 时间进度改为使用当前日期和 target 表 `进量日期` 计算，公式为 `(当前日期-进量日期)/总天数`。
- 网页主表增加 `进量日期` 列，便于直接校验时间进度计算来源。
- 钉钉图片播报使用 markdown 图片消息；如需群成员稳定查看图片，可通过 `DINGTALK_REPORT_BASE_URL` 配置可访问的看板地址。
- 本地看板默认端口从 `8765` 改为 `8766`，并支持通过 `.dashboard-server-disabled` 临时阻止自动启动服务。
- 播报图下载接口在 PNG 已存在时直接返回文件，避免钉钉抓图时触发重生成导致超时或白块。
- 最新期次明细表增加 `下单日期` 列，并将原 `进量日期` 表头改为 `目标进量日期`，避免将 target 进量日期误认为 demo 更新日期。
- 上传 demo/target 后在上传卡片内显示持久的上传中、成功或失败提示，避免只依赖短暂 toast。
- 最新期次明细的状态标签增加颜色区分：正常为绿色、落后为红色、未开单/仅现状为黄色、已完成为蓝色。
- 最新期次明细表不再展示 `下单日期` 列，保留 `目标进量日期` 作为 target 进度口径字段。
- 最新期次明细表不再展示 `目标进量日期` 列，仅保留其作为后端进度计算字段。
- 最新期次明细筛选区增加 `下单日期` 筛选项，选项来自 demo 表中的下单日期。
- 修正时间进度公式，避免小学最新期次因 `target_time - 进量日期 = 6` 全部显示为 `0%`，以及高中负进度被误判为 `100%`。
- 回滚普通看板和学部播报的错误期次口径，恢复以 target 表最新期次为准；`lec1元占比` 固定按小学 `暑_8` 统计，并修正 RQ 成单量为 546。
- 优化筛选区下拉控件 UI，统一输入框、下拉框的边框、聚焦态、悬停态和内置箭头样式。
- 删除侧边栏品牌区下方的 `Summary Monitor · V1` 版本说明。

### Verified

- 已验证 summary 接口可返回最新期次指标。
- 已验证自然语言查询示例可返回对应 `成单量`。
- 已验证播报图下载接口可返回有效 PNG。
- 已验证钉钉播报 payload 生成逻辑，不触发真实群消息。
- 已验证当前 `DINGTALK_REPORT_BASE_URL` 下的小学、初中、高中播报图 URL 均返回 `200 image/png`。
- 已验证 `lec1元占比` 播报图本地下载接口和公网图片 URL 均返回有效 PNG。
